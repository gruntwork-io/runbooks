# gruntbooks open

Use `gruntbooks open` to open a gruntbook in your browser and use it.

This command is intended for **consumers** of gruntbooks.

## Usage

```bash
gruntbooks open GRUNTBOOK_SOURCE [flags]
```

## Arguments

- `GRUNTBOOK_SOURCE` - A local path or remote URL pointing to a `gruntbook.mdx` file, the directory containing a `gruntbook.mdx` file, or an OpenTofu/Terraform module directory.

### Local paths

```bash
gruntbooks open ./path/to/gruntbook
gruntbooks open /absolute/path/to/gruntbook
gruntbooks open ./path/to/gruntbook.mdx
```

### Remote sources

You can open a gruntbook directly from a GitHub or GitLab URL without cloning the repo first:

```bash
# GitHub browser URL (copy/paste from your browser)
gruntbooks open https://github.com/org/repo/tree/main/gruntbooks/setup-vpc

# GitLab browser URL
gruntbooks open https://gitlab.com/org/repo/-/tree/main/gruntbooks/setup-vpc

# OpenTofu/Terraform module source format
gruntbooks open github.com/org/repo//gruntbooks/setup-vpc?ref=v1.0
gruntbooks open "git::https://github.com/org/repo.git//gruntbooks/setup-vpc?ref=main"
```

| Format | Example |
|--------|---------|
| GitHub browser (dir) | `https://github.com/org/repo/tree/main/gruntbooks/setup-vpc` |
| GitHub browser (file) | `https://github.com/org/repo/blob/main/gruntbooks/setup-vpc/gruntbook.mdx` |
| GitLab browser (dir) | `https://gitlab.com/org/repo/-/tree/main/gruntbooks/setup-vpc` |
| GitLab browser (file) | `https://gitlab.com/org/repo/-/blob/main/gruntbooks/setup-vpc/gruntbook.mdx` |
| OpenTofu GitHub shorthand | `github.com/org/repo//path?ref=v1.0` |
| OpenTofu git::https | `git::https://github.com/org/repo.git//path?ref=main` |

<Aside type="note">
When opening a remote gruntbook, a temporary working directory is automatically used (equivalent to `--working-dir=::tmp`). You can override this with `--working-dir` if needed.
</Aside>

### OpenTofu/Terraform modules

If `GRUNTBOOK_SOURCE` points to a directory containing `.tf` files (but no `gruntbook.mdx`), Gruntbooks auto-detects it as an OpenTofu/Terraform module. It generates a temporary gruntbook that parses the module's variables and renders a web form for them.

```bash
# Local module directory
gruntbooks open ./modules/rds

# Remote module via GitHub URL
gruntbooks open https://github.com/org/repo/tree/main/modules/rds

# Remote module via OpenTofu shorthand
gruntbooks open github.com/org/repo//modules/rds?ref=v1.0
```

By default, the auto-generated gruntbook uses the `::terragrunt` template. Use `--tf-gruntbook` to choose a different built-in template or provide your own custom gruntbook. See [Flags](#flags) below.

### Authentication for private repos

For public repositories, no authentication is needed. For private repositories, set one of the following:

- **GitHub:** `GITHUB_TOKEN` or `GH_TOKEN` environment variable, or run `gh auth login`
- **GitLab:** `GITLAB_TOKEN` environment variable, or run `glab auth login`

## Flags

- `--working-dir <path>` - Base directory for script execution and file generation (default: current directory)
  - All relative paths are resolved from this directory
  - Can be absolute or relative to current directory
  - Use `--working-dir=::tmp` for a temporary directory (automatically cleaned up on exit), useful for isolated testing or sandboxed execution
- `--output-path <path>` - Directory where generated files will be written (default: `generated`)
  - Resolved relative to the working directory
- `--tf-gruntbook <keyword-or-path>` - Select a built-in template or provide a local custom gruntbook for OpenTofu/Terraform modules. Built-in templates:
  - `::terragrunt` (default) — Generates a `terragrunt.hcl` file
  - `::terragrunt-github` — Full GitOps workflow: GitHub auth, clone, directory picker, generate `terragrunt.hcl`, and open a pull request
  - `::tofu` — Generates a plain `main.tf` file
  - Or pass a local path to a custom gruntbook directory (e.g., `--tf-gruntbook ./my-gruntbook/`)
  - Remote URLs are not supported — download the template first and pass the local path. This is intentional: a remote `--tf-gruntbook` could silently pull in an untrusted gruntbook that executes scripts on your machine.
- `--port <number>` - Port for the backend + frontend server (default: 7825). Useful when running multiple gruntbooks simultaneously or when the default port is already in use.
- `--no-telemetry` - Disable anonymous telemetry. Can also be set via `GRUNTBOOKS_TELEMETRY_DISABLE=1` environment variable.

## What it does

When you run `gruntbooks open`:

1. **Downloads (if remote)** - If given a remote URL, downloads just the gruntbook directory via sparse git clone
2. **Auto-detects (if TF module)** - If the target is an OpenTofu/Terraform module directory (contains `.tf` files but no `gruntbook.mdx`), generates a temporary gruntbook using the selected template
3. **Starts the Backend Server** - Launches a Go-based HTTP server on `localhost` (port 7825 by default, configurable with `--port`)
4. **Launches the Browser** - Opens your default web browser to the server URL
5. **Serves the Frontend** - The web UI connects to the backend API to process the gruntbook
6. **Keeps Running** - The server continues running until you close the browser or press Ctrl+C

## Troubleshooting

**Port already in use:**
If the default port (7825) is already in use, you'll see an error. Either stop the other process or use `--port` to pick a different port (e.g., `gruntbooks open --port 8080 ./my-gruntbook`).

**Browser doesn't open:**
If the browser doesn't open automatically, you can manually navigate to `http://localhost:7825` (or your custom `--port`) after running the command.

**Browser window with the gruntbook doesn't open:**
Sometimes, your browser seems to launch, but you don't see the Gruntbook. The most likely issue here is that your browser launched with multiple tabs and windows and your Gruntbooks tab is not visible. Either look through all your browser windows, or go directly to http://localhost:7825 (or your custom `--port`).

**Gruntbook not found:**
Make sure the path points to a valid `gruntbook.mdx` file.

**Authentication errors for remote gruntbooks:**
If you see "authentication required", make sure you have set the appropriate token for the git host. See the Authentication section above.
