The Gruntbooks CLI provides several commands for working with gruntbooks. All commands accept a local path or a remote URL as the gruntbook source.

- [`gruntbooks open`](/commands/open) — Open a gruntbook for consumers
- [`gruntbooks serve`](/commands/serve) — Start the backend server (for developers)
- [`gruntbooks watch`](/commands/watch) — Watch a gruntbook for changes (for authors)

### Remote URLs

Every command that accepts a `GRUNTBOOK_SOURCE` can open a gruntbook directly from a remote URL, removing the need to manually download any gruntbook files.

```bash
gruntbooks open https://github.com/org/repo/tree/main/gruntbooks/launch-rds
```

See [`gruntbooks open`](/commands/open/#remote-sources) for supported URL formats and authentication for private repos.
