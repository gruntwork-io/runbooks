# gruntbooks watch

Use `gruntbooks watch` to "watch" a `gruntbook.mdx` file (or its scripts, checks, or template files) for changes and automatically reload the Gruntbook as needed. 

This command is intended for **authors** of gruntbooks.

<Aside type="caution">
When you update a `command` property, local template, or local script, the gruntbook will automatically load these changes in the _UI_, but because of the [executable registry](/security/execution-model/#executable-registry), the gruntbook will continue to execute the _old_ versions of these files. The solution is to always restart your gruntbook (re-run `gruntbooks watch`) when you make a template or script change.
</Aside>

## Usage

```bash
gruntbooks watch GRUNTBOOK_SOURCE [flags]
```

## Arguments

- `GRUNTBOOK_SOURCE` - A local path or remote URL pointing to a `gruntbook.mdx` file, the directory containing a `gruntbook.mdx` file, or an OpenTofu/Terraform module directory. See [gruntbooks open](/commands/open) for supported remote URL formats and OpenTofu/Terraform module auto-detection.

<Aside type="note">
When watching a remote gruntbook, file watching operates on the local downloaded copy. Changes to the remote repository will not be detected automatically.
</Aside>

## Flags

- `--working-dir <path>` - Base directory for script execution and file generation (default: current directory)
  - All relative paths are resolved from this directory
  - Can be absolute or relative to current directory
  - Use `--working-dir=::tmp` for a temporary directory (automatically cleaned up on exit), useful for isolated testing or sandboxed execution
- `--output-path <path>` - Directory where generated files will be written (default: `generated`)
  - Resolved relative to the working directory
- `--tf-gruntbook <keyword-or-path>` - Select a built-in template or provide a local custom gruntbook for OpenTofu/Terraform modules. Remote URLs are not supported. See [gruntbooks open](/commands/open/#flags) for available templates and details.
- `--port <number>` - Port for the backend + frontend server (default: 7825). Useful when running multiple gruntbooks simultaneously or when the default port is already in use.
- `--disable-live-file-reload` - Enable executable registry validation. Scripts will not reload from disk when they are updated; instead, you must re-run the `gruntbooks` command to apply script changes. This trades lower convenience for higher security. See [Execution Security Model](/security/execution-model) for details.
- `--no-telemetry` - Disable anonymous telemetry. Can also be set via `GRUNTBOOKS_TELEMETRY_DISABLE=1` environment variable.

## What it does

When you run `gruntbooks watch`:

1. **Starts the Backend Server** - Launches a Go-based HTTP server on `localhost` (port 7825 by default, configurable with `--port`)
2. **Opens Your Browser** - Automatically navigates to the server URL
3. **Watches for Changes** - Monitors the gruntbook file for any modifications
4. **Auto-Reloads** - Automatically refreshes the browser when changes are detected (within ~300ms)

### Writing a new gruntbook

```bash
gruntbooks watch /path/to/gruntbook
```

Then in your editor:
1. Make changes to `gruntbook.mdx`
2. Save the file
3. See your changes instantly in the browser - no manual refresh needed!

## Technical details

### File watching

- Uses `fsnotify` for efficient file system monitoring
- Watches the directory containing your gruntbook file
- Implements debouncing (300ms) to handle editors that save files multiple times
- Only triggers on Write and Create events for your specific gruntbook file

### Auto-reload mechanism

- Uses Server-Sent Events (SSE) to push notifications from server to browser
- The browser maintains a persistent connection to `/api/watch/sse`
- When the file changes, the server sends a `file-change` event
- The browser receives the event and automatically reloads the page
